# 微信小程序文件收取系统产品需求文档(PRD)

## 1. 产品概述

### 1.1 产品定位
微信小程序文件收取系统是一款专为教育场景设计的轻量级文件管理工具，旨在简化师生间的文件发布、提交和管理流程。

### 1.2 核心价值
- **提升效率**：摒弃传统纸质文件收发模式，实现数字化文件管理
- **降低成本**：减少纸质打印成本，节约教学资源
- **便捷管理**：教师可随时随地发布文件任务，学生可及时查看和提交
- **过程透明**：实时跟踪文件提交状态，避免遗漏
- **数据留存**：文件记录可长期保存，便于教学档案管理

### 1.3 产品特色
- 基于微信生态，无需额外下载APP，使用门槛低
- 专注文件收取，功能精准聚焦
- 师生双端设计，满足不同角色需求
- 简洁易用的界面设计，适合各年龄段用户

## 2. 用户需求分析

### 2.1 目标人群

#### 2.1.1 主要用户
- **教师用户**：中小学、高等院校的任课教师
  - 年龄段：25-55岁
  - 技术水平：中等，熟悉微信基本操作
  - 使用频率：工作日频繁使用

- **学生用户**：各级在校学生
  - 年龄段：6-25岁
  - 技术水平：较高，对移动应用操作熟练
  - 使用频率：根据文件任务安排不定期使用

#### 2.1.2 潜在用户
- 培训机构讲师
- 企业内训师
- 家教老师

### 2.2 使用场景

#### 2.2.1 教师端使用场景
- **课后发布文件任务**：课程结束后，教师需要快速发布文件收取任务
- **文件收集统计**：临近截止时间，教师需要查看提交情况
- **学生管理维护**：新学期开始，需要添加或调整学生名单
- **移动办公**：在通勤途中或非办公时间处理文件相关事务

#### 2.2.2 学生端使用场景
- **查看文件任务**：课后或回家后查看老师发布的文件收取任务
- **准备和提交文件**：准备文件后进行上传提交
- **确认提交状态**：确保文件已成功提交，避免遗漏
- **查看历史记录**：回顾之前的文件提交情况

### 2.3 用户痛点

#### 2.3.1 教师痛点
- **收集困难**：传统方式下收集文件耗时耗力，容易遗漏
- **统计麻烦**：难以快速统计文件提交情况和完成率
- **存储问题**：纸质文件占用空间大，电子文件分散在各种平台
- **时效性差**：无法实时了解学生提交进度
- **管理混乱**：多个班级、多门课程的文件管理复杂

#### 2.3.2 学生痛点
- **忘记提交**：缺乏统一的文件提醒和管理工具
- **提交不便**：需要通过多种渠道提交文件，操作繁琐
- **状态不明**：不确定文件是否成功提交
- **查找困难**：难以快速找到特定文件任务的要求和截止时间
- **重复操作**：每次提交都需要重新输入课程、文件任务信息

## 3. 核心功能

### 3.1 学生端功能

#### 3.1.1 文件任务列表管理
- **待完成文件任务列表**
  - 显示所有未提交的文件任务
  - 按截止时间排序，紧急文件任务置顶
  - 显示文件任务标题、截止时间、发布教师
  - 支持筛选功能（按教师、时间范围）
  - 点击单个文件任务后，进入文件任务详情页

- **已完成文件任务列表**
  - 显示已提交的文件任务历史记录
  - 显示提交时间、提交状态
  - 支持重新查看已提交的文件

- **文件任务详情**
  - 显示该文件任务对应所有学生的提交状态，提交时间
  - 自己的提交记录可以进行修改，修改提交的文件

#### 3.1.2 文件提交功能
- **文件上传**
  - 支持多种文件格式（PDF、Word、图片、压缩包等）
  - 支持多文件批量上传
  - 文件大小限制提醒
  - 上传进度显示

- **提交确认**
  - 提交前预览功能
  - 提交成功确认通知
  - 支持提交后修改（在截止时间前）
  - 支持在小程序中修改文件名
  - 支持一键复制文件名模板

#### 3.1.3 个人中心
- 个人信息展示（姓名、班级、学号）
- 提交统计（总数、按时率等）
- 设置功能（暗黑模式）

### 3.2 教师端功能

#### 3.2.1 学生管理
- **学生列表管理**
  - 添加/删除学生
  - 批量导入学生信息
  - 学生分组管理（按班级、入学年份）
  - 学生信息编辑


#### 3.2.2 文件任务发布
- **文件任务创建**
  - 文件任务标题和详细描述
  - 截止时间设置
  - 目标学生/班级选择
  - 文件格式要求说明
  - 文件任务模板保存和复用
  - 文件名模板可以设置保存和复用

- **文件任务管理**
  - 已发布文件任务列表
  - 文件任务编辑和删除
  - 文件任务状态管理（草稿、已发布、已截止）

#### 3.2.3 提交情况查看
- **统计概览**
  - 总体提交率统计
  - 按时提交率分析
  - 图表展示提交趋势

- **详细列表**
  - 学生提交状态一览
  - 未提交学生名单
  - 文件下载和预览
  - 所有文件一键下载
  - 提交时间记录

#### 3.2.4 消息通知
- **提醒功能**
  - 自动提醒未提交学生
  - 截止时间临近提醒
  - 提交状态变更通知

### 3.3 共通功能

#### 3.3.1 身份认证
- 用户账号不可以由用户主动创建，只可以在教师端人员管理处进行导入。
- 微信登录授权
- 输入账号/手机号
- 输入密码
- 用户角色识别（学生/教师）
- 权限控制

#### 3.3.2 消息通知
- 微信模板消息推送
- 文件任务相关提醒
- 系统通知


## 4. 关键业务流程

### 4.1 教师发布文件任务流程

```
1. 教师登录系统
   ↓
2. 选择"发布文件任务"功能
   ↓
3. 填写文件任务信息
   - 文件任务标题
   - 文件任务描述（可选）
   - 开始时间
   - 截止时间
   - 文件格式要求（可选）
   - 文件名模板（可选）
   ↓
4. 选择目标学生/班级
   ↓
5. 预览文件任务内容
   ↓
6. 确认发布
   ↓
7. 系统推送通知给学生
   ↓
8. 文件任务进入已发布列表
```

### 4.2 学生提交文件流程

```
1. 学生登录系统
   ↓
2. 查看待完成文件任务列表
   ↓
3. 选择要提交的文件任务
   ↓
4. 查看文件任务详情和要求
   ↓
5. 上传文件
   - 选择文件
   - 检查格式和大小
   - 确认上传
   ↓
6. 预览提交内容
   ↓
7. 确认提交
   ↓
8. 系统确认提交成功
   ↓
9. 文件任务移至已完成列表
   ↓
10. 教师收到提交通知（可选）
```

### 4.3 教师查看提交情况流程

```
1. 教师登录系统
   ↓
2. 选择"查看提交情况"
   ↓
3. 选择要查看的文件任务
   ↓
4. 查看统计概览
   - 总提交数
   - 提交率
   - 未提交学生名单
   ↓
5. 查看详细列表
   - 每个学生的提交状态
   - 提交时间
   - 下载文件查看
   ↓
6. （可选）发送提醒给未提交学生
```

### 4.4 用户注册和身份认证流程

```
1. 用户首次进入小程序
   ↓
2. 微信授权登录
   ↓
3. 输入学号（对应学生）/手机号（对应教师）
   ↓
4. 输入密码
   ↓
5. 正式使用
```

### 4.5 异常处理流程

#### 4.5.1 文件上传失败处理
```
文件上传 → 检查网络连接 → 检查文件格式 → 检查文件大小 → 重新上传 → 联系技术支持
```

#### 4.5.2 文件任务截止时间处理
```
截止时间到达 → 系统自动关闭提交 → 发送截止通知 → 教师可选择延期 → 重新开放提交
```

#### 4.5.3 数据同步异常处理
```
数据同步失败 → 本地缓存保存 → 网络恢复后重试 → 数据完整性检查 → 同步完成确认
```

## 5. 非功能性需求

### 5.1 性能要求
- 页面加载时间不超过3秒
- 文件上传支持断点续传

### 5.2 兼容性要求
- 支持微信最近3个版本
- 适配主流手机型号
- 网络环境适应性强
